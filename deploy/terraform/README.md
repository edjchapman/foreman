# Terraform — Foreman platform on Railway

Declares everything the community Railway provider can express: the project,
the five services (Postgres, Redis, web, worker, beat — databases are plain
image + volume services, which is all Railway's "templates" are), every
environment variable (secrets generated by `random_password`), and the public
domain. Rationale and the full deploy story: [ADR 0005](../../docs/adr/0005-deployment-platform.md)
and [docs/deploy.md](../../docs/deploy.md).

## Honest limitations

The provider (v0.6.x) cannot express three settings, and Railway's
`railway.json` config-as-code doesn't apply to image-sourced services either —
so after the first `apply` they are set **once** in the dashboard (printed by
`terraform output manual_steps`):

| Service | Setting | Value |
|---|---|---|
| web | Pre-Deploy Command | `uv run --no-dev python manage.py migrate` |
| web | Healthcheck Path | `/readyz` |
| worker | Custom Start Command | `uv run --no-dev celery -A config worker -l info --concurrency 2` |
| beat | Custom Start Command | `uv run --no-dev celery -A config beat -l info` |

They survive image re-pins by the CD script (deploys change `source.image`,
not service settings), so this is genuinely one-time per platform build.

## Usage

```bash
export RAILWAY_TOKEN=<ACCOUNT token>   # workspace-scoped: Terraform creates the project.
                                       # (CD uses a narrower PROJECT token — different secret.)
terraform init
terraform apply                        # optionally: -var app_version=0.7.0
terraform output manual_steps          # dashboard one-offs, then set the CI variables:
terraform output github_ci_variables
```

Region: the provider's `regions` attribute mishandles conditional values
(v0.6.x), so region is deliberately not declared here — set the workspace's
default region (e.g. EU West) in Railway account settings *before* the first
`apply`.

## The off switch

The demo is designed to be torn down when not in use — all data regenerates
from the sample-import button:

```bash
terraform destroy   # → billing drops to the Hobby subscription floor (~$5/mo)
terraform apply     # → platform back in minutes (re-run manual_steps + CI vars: IDs change)
```

## State

State is **local and git-ignored** (`.gitignore` here): it contains the
generated DB passwords and `DJANGO_SECRET_KEY`. Single-operator portfolio
setup — no remote backend. If you ever need a value: `terraform output`, or
read it in the Railway dashboard. Losing state entirely is recoverable:
`destroy` via the dashboard and re-`apply` from scratch.
